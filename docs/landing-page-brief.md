# Бриф: лендинг Startup Hunt

Документ для AI-агента, который создаёт публичную лендинг-страницу. С лендинга пользователь переходит в само приложение (регистрация / вход).

---

## 1. Продукт в одном предложении

**Startup Hunt** — минималистичная платформа в духе Product Hunt, где основатели публикуют карточки стартапов, а эксперты-оценщики голосуют и оставляют обратную связь; рейтинг формируется открыто и в реальном времени.

---

## 2. Краткое описание (для hero-блока)

Startup Hunt помогает командам на ранней стадии показать свой проект закрытому кругу оценщиков, собрать голоса и комментарии и понять, какие идеи действительно резонируют. Без шума публичных площадок — только структурированная подача, файлы, рейтинг и диалог между основателем и экспертом.

**Альтернативный подзаголовок (короче):**  
Подавайте стартапы. Получайте честные оценки. Смотрите, кто в топе.

---

## 3. Целевая аудитория

| Кто | Зачем приходит |
|-----|----------------|
| **Основатели (founder)** | Опубликовать карточку стартапа с описанием и материалами (презентация, скриншоты, архив) |
| **Оценщики (evaluator)** | Просматривать заявки, голосовать ▲/▼, оставлять комментарии |
| **Администратор** | Назначать роли участникам закрытого сообщества |

Регистрация открыта, но полный доступ выдаёт администратор после модерации роли.

---

## 4. Ключевые преимущества (для секции «Почему Startup Hunt»)

1. **Прозрачный рейтинг** — один голос на стартап, можно изменить; итоговый score виден всем участникам.
2. **Всё в одной карточке** — название, описание, вложения (PDF, изображения, ZIP, DOCX).
3. **Роли под задачу** — основатель публикует, оценщик голосует, админ управляет доступом.
4. **Живой диалог** — комментарии от оценщиков и ответ основателя прямо на странице стартапа.
5. **Удобная таблица** — поиск, сортировка по рейтингу и дате, мобильная вёрстка.

---

## 5. Как это работает (пошагово)

Использовать как секцию «Как это работает» — 3–4 шага с иконками или иллюстрациями.

### Для основателя
1. **Регистрация** — создайте аккаунт; администратор назначит роль «Основатель».
2. **Создайте карточку** — название, описание, до 5 файлов (до 10 МБ каждый).
3. **Получайте оценки** — смотрите рейтинг в общей таблице и отвечайте на комментарии.

### Для оценщика
1. **Регистрация** — после назначения роли «Оценщик» открывается доступ к каталогу.
2. **Изучайте стартапы** — фильтруйте, сортируйте, открывайте детальные страницы.
3. **Голосуйте и комментируйте** — ▲ или ▼ влияет на рейтинг; один голос на карточку.

### Для администратора
- Назначает роли в панели `/admin/users` — founder, evaluator или ожидание.

## 6. Призывы к действию (CTA)

| Место | Текст кнопки | Куда ведёт |
|-------|--------------|------------|
| Hero (главный) | **Начать** / **Зарегистрироваться** | `/register` |
| Hero (вторичный) | **Войти** | `/login` |
| После блока «Как работает» | **Подать стартап** | `/register` |
| Footer | **Войти в платформу** | `/login` |

После входа авторизованные пользователи попадают в приложение: **`/startups`** (таблица стартапов).

> Лендинг — публичная точка входа. Само приложение за auth: без логина `/startups` недоступен.

## 7. Визуальная концепция

### Настроение
- Чистый, уверенный, «product-led» — не корпоративный SaaS, а community-driven discovery.
- Светлая тема, много воздуха, акцент на рейтингах и карточках стартапов.
- Референс по духу: Product Hunt, но проще и приватнее.

### Цвета (из дизайн-системы приложения)

| Токен | HEX | Использование |
|-------|-----|---------------|
| Primary | `#FF6154` | CTA-кнопки, акценты, upvote |
| Primary hover | `#E5564A` | hover на кнопках |
| Background | `#FAFAFA` | фон страницы |
| Surface | `#FFFFFF` | карточки, блоки |
| Text | `#1A1A1A` | заголовки и основной текст |
| Text muted | `#6B7280` | подписи, вторичный текст |
| Border | `#E5E7EB` | границы карточек |
| Success | `#16A34A` | позитивные акценты |

### Типографика
- Шрифт: **Inter**, fallback system-ui, sans-serif
- Заголовок hero: 40–56px, font-weight 600–700
- Подзаголовок: 18–20px, muted
- Body: 16px

### Визуальные элементы для лендинга
- **Hero-иллюстрация / mockup:** стилизованная таблица стартапов с колонками «Рейтинг | Название | Автор» и кнопками ▲▼ — главный визуальный якорь.
- **Мини-карточки** в секции features: иконка + заголовок + 1 строка текста.
- **Схема потока:** Founder → Card → Votes → Leaderboard (можно простая горизонтальная timeline).
- Лёгкие тени: `0 4px 6px rgba(0,0,0,0.07)` на карточках.
- Скругления: 8–12px.

### Чего избегать
- Тёмная «крипто»-эстетика, неон, градиенты ради градиентов
- Stock-фото людей в переговорках
- Перегруженный текст — лендинг сканируется за 5 секунд

```
  «Закрытое сообщество основателей и оценщиков»

  3 шага с иконками (см. §5)

[Для кого]
  3 колонки: Основатель | Оценщик | Администратор

  4–5 пунктов (см. §4)

[Финальный CTA]
  H2: Готовы показать свой стартап?
  Button → /register

[Footer]
  Startup Hunt © 2026
  Войти · Регистрация
```

## 9. Тексты для копирайтинга (готовые формулировки)

### Hero
- **H1:** Откройте лучшие стартапы. Или покажите свой.
- **Sub:** Startup Hunt — платформа, где основатели публикуют идеи, а эксперты честно голосуют. Прозрачный рейтинг, файлы и обратная связь в одном месте.

### Features (заголовки)
- Прозрачный рейтинг
- Материалы к каждой заявке
- Роли для вашего сценария
- Комментарии и ответы
- Умная таблица с поиском

### Финальный CTA
- **H2:** Присоединяйтесь к Startup Hunt
- **Sub:** Зарегистрируйтесь — администратор откроет доступ к вашей роли.
- **Button:** Создать аккаунт

## 10. Технические требования к лендингу

| Параметр | Значение |
|----------|----------|
| Язык интерфейса | Русский |
| `lang` атрибут | `ru` |
| Адаптивность | Mobile-first, breakpoints 768px / 1024px |
| Доступность | Семантический HTML, контраст WCAG AA, focus states на кнопках |
| Производительность | Без тяжёлых анимаций; предпочтительно CSS, не видео-фон |
| SEO | title: «Startup Hunt — платформа оценки стартапов»; meta description из §2 |

### Интеграция с приложением

Варианты реализации (на выбор разработчика):

1. **Страница в том же Next.js-проекте** — заменить `app/page.tsx` (сейчас редирект на `/startups`) на публичный лендинг; авторизованных по-прежнему редиректить на `/startups`.
2. **Отдельный статический HTML** — с абсолютными ссылками на домен приложения.

Маршруты приложения:
- `/register` — регистрация
- `/login` — вход
- `/startups` — главный экран (после входа)
- `/startups/new` — создание стартапа (founder/admin)
- `/admin/users` — управление ролями (admin)

## 11. Лучшие практики лендинга (чеклист для агента)

- [ ] Один главный CTA above the fold — «Зарегистрироваться»
- [ ] Hero отвечает на: **что это**, **для кого**, **что делать дальше** — за 5 секунд
- [ ] Визуал показывает продукт (mockup таблицы), а не абстрактную «инновацию»
- [ ] Секция «Как работает» — не больше 3–4 шагов
- [ ] Повтор CTA внизу страницы
- [ ] Navbar липкий (sticky), CTA всегда доступен
- [ ] Мобильная версия: кнопки full-width, текст не мельче 16px
- [ ] Нет ложных обещаний («тысячи пользователей») — MVP, закрытое сообщество
- [ ] Единая цветовая система с приложением (`#FF6154` primary)

---

## 12. Что НЕ показывать на лендинге

- Внутренние детали стека (Next.js, Prisma, SQLite)
- Учётные данные админа
- Функции «в планах», которых нет: OAuth, email-уведомления, теги, публичный каталог без авторизации
