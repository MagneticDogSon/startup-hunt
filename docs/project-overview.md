# Startup Hunt — Project Overview

## Название

Startup Hunt (рабочее название)

## Цель MVP

Минимальная платформа в духе Product Hunt: основатели подают карточки стартапов с описанием и файлами, оценщики голосуют вверх/вниз, все видят таблицу с простыми фильтрами.

## Пользователи и роли

| Роль | Описание |
|------|----------|
| `pending` | Зарегистрирован, но без прав до назначения админом |
| `founder` | Создаёт и редактирует свои карточки стартапов |
| `evaluator` | Просматривает карточки и голосует (▲/▼) |
| `admin` | Назначает роли, полный доступ к карточкам |

Регистрация общая; роль назначает админ.

## In scope (MVP)

- Регистрация и логин (email + пароль)
- CRUD карточек стартапа (founder — свои, admin — все)
- Загрузка файлов к карточке (до 5 файлов, 10 MB каждый)
- Голосование ±1 (один голос на карточку, можно сменить)
- Таблица стартапов с рейтингом
- Фильтры: поиск по тексту, сортировка по рейтингу/дате
- Админ-панель назначения ролей

## Out of scope (MVP)

- Комментарии
- Теги и категории
- Email-уведомления
- OAuth (Google, GitHub)
- Продвинутая аналитика
- Модерация контента
- Пагинация (до ~100 записей достаточно)

## Критерии готовности MVP

1. Основатель может создать карточку с описанием и прикрепить файлы
2. Оценщик может проголосовать вверх или вниз; рейтинг обновляется
3. Таблица показывает все карточки с фильтрами и сортировкой
4. Админ может назначить роль любому зарегистрированному пользователю
